FastAPI 自动生成 API 文档

对话背景

用户提问：API 文档是如何生成的？

核心结论：FastAPI 自动生成 API 文档，开发者不需要手动编写任何文档文件。

一、自动生成的原理

FastAPI 在启动时扫描所有路由定义，基于以下三要素构建 OpenAPI 规范（JSON），再由 Swagger UI 或 ReDoc 渲染为可视化页面：

信息来源说明生成文档中的内容
路由装饰器
@router.post("/search")		HTTP 方法 + URL 路径接口地址和请求方法（如 POST /api/companies/search）
Pydantic 模型
参数类型和 response_model		字段名、类型、是否必填、默认值请求体和响应体的完整 Schema
FastAPI 参数
Query()、Path() 等		查询参数、路径参数的元信息参数的类型、必填、示例值

二、代码与文档的对应关系

以搜索接口为例，app/routers/companies.py 中的函数定义：

@router.post("/search", response_model=SearchResponse)
async def search_companies_api(
    req: SearchRequest,
    db: Session = Depends(get_db)
):
    """Search companies by keyword across enabled data sources."""
    ...

FastAPI 自动提取的信息如下：

代码元素提取信息文档展示
@router.post("/search")POST + /api/companies/search接口地址
req: SearchRequest请求体模型：keyword(str, required)、filters(dict, optional)请求体 Schema
response_model=SearchResponse响应体模型：total(int)、companies(array)响应体 Schema
函数文档字符串"Search companies by keyword..."接口描述

三、访问地址

启动服务后，FastAPI 自动挂载了两个文档页面（无需任何额外配置）：

地址引擎特点
http://localhost:8000/docsSwagger UI可视化，可直接点击 "Try it out" 发送请求，在线调试接口
http://localhost:8000/redocReDoc更美观的静态文档排版，适合阅读和分享
关键优势：接口代码一旦修改（增删字段、改路径、换方法），文档自动同步更新。不需要你写一行文档代码，也不需要维护单独的 YAML/JSON 文件。

四、Pydantic 模型的自动序列化

app/schemas.py 中定义的模型是文档的核心数据源：

class SearchRequest(BaseModel):
    keyword: str
    filters: Optional[dict] = None

class SearchResponse(BaseModel):
    total: int
    companies: list[CompanyBrief]

FastAPI 会将这些类型注解翻译为 JSON Schema：

SearchRequest:
  type: object
  properties:
    keyword:
      type: string
    filters:
      type: object
  required: ["keyword"]

SearchResponse:
  type: object
  properties:
    total:
      type: integer
    companies:
      type: array
      items:
        $ref: "#/components/schemas/CompanyBrief"
  required: ["total", "companies"]

这个 JSON Schema 就是 Swagger UI 和 ReDoc 渲染文档的基础数据。
