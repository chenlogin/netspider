1. 产品概述

NetSpider 是一款面向公网的企业信息采集工具。用户通过输入搜索关键词或筛选条件，系统自动在互联网上检索符合条件的公司列表，并支持对目标公司进行深度信息抓取，获取企业详情。

1.1 核心场景

  1. 公司发现：根据用户输入的条件（行业、关键词、地域等），在公开网站上检索并汇总公司列表。
  2. 详情抓取：用户在搜索结果中选择某家公司，系统自动抓取该公司的详细信息（工商信息、联系方式、业务描述等）。

2. 用户角色

角色描述核心诉求
普通用户使用系统检索公司信息快速找到目标公司，获取完整信息
管理员管理爬虫配置、数据源、调度策略灵活管理爬取源，监控爬取状态

3. 功能需求

3.1 公司列表检索

字段说明
功能编号F-001
优先级P0
描述接收用户输入的搜索条件，在多个公开数据源中检索公司并返回列表
输入关键词（公司名称/行业/业务描述）、可选的地域筛选
输出匹配的公司列表，包含：公司名称、统一社会信用代码、注册地址、简要行业、数据来源

业务流程：

  1. 用户在界面输入关键词及可选筛选条件
  2. 系统将关键词转化为各数据源对应的搜索请求
  3. 并行查询多个公开数据源
  4. 对返回结果进行去重、排序、分页
  5. 将公司列表返回前端展示

3.2 公司详情抓取

字段说明
功能编号F-002
优先级P0
描述用户在列表中选中一家公司，系统自动抓取并展示该公司的详细信息
输入公司标识（统一社会信用代码或数据源返回的唯一 ID）
输出公司详细信息

详情字段包括但不限于：

  • 基本信息：公司名称、简称、统一社会信用代码、法定代表人、注册资本、成立日期、营业期限
  • 联系信息：注册地址、办公地址、联系电话、邮箱、官网
  • 经营信息：经营范围、行业分类、企业状态（存续/注销/吊销）
  • 股东信息：股东名称、出资额、持股比例
  • 变更记录：工商变更历史
  • 数据来源及抓取时间

业务流程：

  1. 用户在列表页面点击目标公司
  2. 系统根据数据来源构造详情页抓取请求
  3. 模拟访问目标页面，解析结构化数据
  4. 清洗、校验、存储抓取结果
  5. 将详情数据返回前端展示

3.3 数据源管理

字段说明
功能编号F-003
优先级P1
描述管理可配置的公开数据源，支持增删改查和启停控制

3.4 结果导出

字段说明
功能编号F-004
优先级P1
描述支持将搜索结果和详情数据导出为 CSV / Excel / JSON 文件

3.5 爬取日志与监控

字段说明
功能编号F-005
优先级P2
描述记录每次爬取任务的状态、耗时、成功/失败情况，提供简单看板

5.2 技术选型建议

层次技术方案说明
前端HTML/CSS/JS 或 Streamlit轻量级界面，快速迭代
后端框架FastAPI (Python)异步支持好，与爬虫生态契合
爬虫核心Playwright / httpx + BeautifulSoupPlaywright 处理动态渲染页面，httpx 处理静态页面
数据存储SQLiteMVP 阶段轻量持久化
缓存内存缓存 / 可选 Redis减少重复请求，降低被封禁风险
任务调度asyncio并发请求，提高爬取效率

6. 非功能需求

维度要求
性能公司列表搜索响应时间 < 10 秒；详情页抓取 < 15 秒
并发支持同一用户并发 3 个数据源查询
可用性单个数据源失败不影响其他数据源返回结果（降级策略）
合规性遵守目标网站 robots.txt；请求频率可控；不抓取隐私敏感数据
可扩展数据源采用插件化设计，新增数据源只需添加对应适配器

8. 接口设计

8.1 搜索公司列表

POST /api/companies/search
请求体{ "keyword": "string", "filters": { "region": "string" } }
响应体{ "total": int, "companies": [ { "id": int, "name": str, "credit_code": str, "source": str, ... } ] }

8.2 获取公司详情

GET /api/companies/{id}/detail
路径参数id: 公司 ID
响应体{ "id": int, "name": str, "legal_representative": str, "registered_capital": str, ..., "source_url": str }

8.3 导出结果

GET /api/companies/export
查询参数format: csv | json | xlsx；keyword: string（可选）
响应文件下载

9. 项目规划

9.1 MVP（V1.0）交付范围

  • 实现 F-001：基于至少一个数据源的公司列表搜索
  • 实现 F-002：选定公司后的详情抓取与展示
  • 基础前端界面：搜索框 + 结果列表 + 详情面板
  • SQLite 持久化存储

9.2 V1.1 迭代

  • 多数据源并行搜索 + 结果去重合并
  • F-003 数据源管理（增删改查、启停）
  • F-004 结果导出（CSV / Excel）

9.3 V1.2 迭代

  • F-005 爬取日志与监控看板
  • 反爬策略优化（代理池、请求频率控制）
  • 缓存策略优化

10. 风险与约束

风险影响缓解措施
目标网站反爬升级爬取失败或 IP 被封请求限流、User-Agent 轮换、代理池
网页结构变更解析失败、数据缺失解析层与爬取层解耦；定期校验；告警通知
法律合规风险数据使用受限仅抓取公开信息；遵守 robots.txt；控制请求频率
数据准确性不同数据源信息不一致多源交叉验证；标记置信度
注意：本项目仅针对互联网公开信息进行采集，需严格遵守各数据源的服务条款和 robots.txt 协议。严禁采集个人隐私数据或绕过身份验证机制。
